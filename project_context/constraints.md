# Constraints

## 문서 목적
이 문서는 approval-factory MVP의 **필수 제약과 금지 범위**를 정의한다.

## MVP 집중 범위
이 프로젝트는 플랫폼 전체가 아니라 **platform-ready factory MVP**를 만드는 데 집중한다.

## 필수 제약
- 승인자 중심 운영이어야 한다.
- 설계 문서와 계약 문서가 먼저 존재해야 한다.
- 모든 실행은 PR 단위로 쪼개져야 한다.
- evidence 기반 승인 흐름이 있어야 한다.
- 설계 변경 시 관련 문서와 ADR 갱신 여부를 판정해야 한다.

## 금지 사항
- 처음부터 중앙 control plane 설계로 커지기
- 웹 UI부터 만들기
- 멀티 프로젝트/멀티 레포를 먼저 구현하기
- 승인 없는 구조 변경
- 거대한 한 번짜리 구현 PR 만들기
- 문서 없이 코드부터 만드는 흐름

## 유예 사항
다음은 플랫폼 단계로 미룬다.
- 중앙 승인 대시보드
- 사용자/권한 관리
- 조직 정책 엔진
- 관측성 대시보드
- 다중 저장소 운영
- 배포 자동화

## 수동 승인 필수 항목
- 신규 Work Item 범위 확정
- 아키텍처 변경
- 범위 확대
- 보안/비용/성능 예외
- merge 승인
- release 승인

## 운영 제약
- 처음에는 순차 실행을 기본으로 한다.
- 한 번에 한 PR만 구현한다.
- 리뷰와 QA는 구현과 분리한다.
